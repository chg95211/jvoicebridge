Contributing to jVoiceBridge

The jVoiceBridge community welcomes contributions. The source code for jVoiceBridge is available to anyone to use according to the terms of the GNU General Public License (GPL v. 2.0). Development of the project happens in public and is open to individuals, educational institutions, non-profits, and commercial companies. Everyone benefits when fixes or enhancements are implemented and shared with the community.

Contributor Agreement

All contributors of source code, object code, documentation, or other materials must complete a Sun Contributor Agreement (SCA) form before any contributions can be included in the project. Print, complete, sign, scan and email the form to us:

    Sun Contributor Agreement form - Email to jVoiceBridge-sca@sun.com

The SCA FAQ provides more information about this form and the Contributor List includes all individuals and organizations who have signed and submitted SCA forms.

How You Can Contribute

Here are some of the ways you can contribute to jVoiceBridge:

  • Try out jVoiceBridge
    You can download jVoiceBridge from the documents and files page.

  • Participate on the Mailing List
    The mailing lists provide a way for people using the software to benefit from each other's expertise. To really be useful, these lists need your input! Please don't be afraid to post: help out others when you can, post your questions, comments and ideas as you see fit.

  • File a Bug Report
    Bug reports are very helpful to developers, and a great way to contribute to a project. Please use the Issue Tracker to file bugs (defects). Remember to take a little time when filing a bug report; make sure you can reproduce the problem, and be very explicit in your description of the problem.

  • File an Enhancement Request
    If there's a feature you'd like to see and don't, or something you can imagine improving, let us know by filing an enhancement request. Use the Issue Tracker and file an "Enhancement" rather than a "Defect".

  • Contributing Web Content, Documentation, or Translations
    We welcome help in keeping the jVoiceBridge web content up-to-date. The easiest way to contribute content is to add to the Wiki. This is a light-weight process, but please send us a signed Sun Contributor Agreement form first. If you wish to add to, update, or translate the HTML content on the web site, please follow the procedure below for contributing code.

  • Contributing Code to jVoiceBridge
    If there's a feature you'd like to see, or a bug you'd like to see fixed, you have the power to make that happen. In order to maintain the stability of the jVoiceBridge codebase, however, we have a contribution process that everyone, including Sun internal contributors, must follow.

    1. Complete a Sun Contributor Agreement form and mail it back to us at jVoiceBridge-sca@sun.com. If you have already completed one of these forms as part of another open source project, please email us a pointer to that prior form. A signed agreement form is a legal requirement that must be met before we are permitted to accept any contributions.

    2. Use the Issue Tracker to file an Enhancement or Defect that describes the feature you plan to create or the bug you plan to fix, if an entry does not already exist for that feature or bug.

    3. Write your code in your own workspace.

    4. Return to your Enhancement or Defect in the Issue Tracker and submit a patch as an attachment. Click "Create a New Attachment" to attach a diff between your workspace and the jVoiceBridge workspace of the day you add the attachment. In the "Additional comments" section, please include the following information:

      • Date that the attachment was added
      • An explanation of your enhancement
      • Some details on how it works

    5. Post a message to the dev@jvoicebridge.dev.java.net mailing list letting us know that you have added a patch. At this point, a core team member will be assigned to review your contribution. You will be given feedback on what changes, if any, are needed before the code can be accepted. Once a decision has been made to accept the contribution, the assigned core team member will integrate the patch.

    6. Frequent contributors or sub-system owners will be invited to have Developer access to the workspace and to review code contributions from others.
